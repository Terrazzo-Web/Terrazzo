# Terrazzo Terminal

Terrazzo Terminal is a browser-based developer workspace built in Rust and
WebAssembly with the [Terrazzo](https://docs.rs/terrazzo) UI framework. It pairs
a server process with a WASM client so a browser tab can host terminals, file
editing, conversion tools, and port-forward management.

## Features

- **Terminal**: PTY-backed terminal sessions streamed between the browser and
  server. Terminals support multiple tabs, tab titles, resize handling, input
  buffering, reconnect behavior, and close/list/write/resize operations.
- **Text editor**: remote-aware folder browser and file editor. It supports text
  editing, synchronized save state, file watching, search, side-view history,
  file diffs, PDF preview, folder browsing, creating entries, deleting entries,
  download actions, and drag-and-drop moves in the folder view.
- **Converter**: paste text and get useful derived representations. Recognizers
  include X509 PEM, JWT, Base64, JSON, YAML, escaped/unescaped text, TLS info,
  DNS output, and timestamps.
- **Port forwarding**: define port-forward rules for the selected remote and
  track whether each rule is pending, up, offline, or failed.
- **Tiles**: organize the workspace into split panes. The app menu can switch a
  tile between Terminal, Text editor, Converter, and Port forward, split the
  current tile horizontally or vertically, or close a tile.
- **Remotes and persisted state**: selected remotes, tile state, editor paths,
  converter input, and related UI state are kept through the app's state layer.

## Running with Cargo

Prerequisites:

```sh
cargo install --locked wasm-pack
cargo install --locked terrazzo-css-cli
```

Build and run a production-style server:

```sh
cargo run --locked --bin terrazzo-terminal --release --features prod
```

Open the address printed by the process, for example:

```text
Listening on http://127.0.0.1:3001
```

Install the binary from crates.io:

```sh
cargo install --locked terrazzo-terminal --features prod
terrazzo-terminal
```

Useful Cargo validation commands, matching the merge-validation workflow:

```sh
cargo build --bins --features=server,server-all,max_level_debug,debug,diagnostics
cargo build --bins --features=server,server-all,max_level_info --release
RUSTFLAGS="-A unused-crate-dependencies" cargo test --workspace --all-features
```

## Building with Bazel

The Bazel build creates separate client WASM and server binary targets. The main
production and debug targets are generated by `terminal_rules` in
`terminal/BUILD.bazel`.

Common targets:

```sh
bazel build //terminal:server
bazel build //terminal:server-debug
bazel run //terminal:server-debug
```

Feature-specific server targets are also generated:

```sh
bazel build //terminal:terminal-server-debug
bazel build //terminal:text-editor-server-debug
bazel build //terminal:converter-server-debug
```

Run the normal Bazel validation set:

```sh
bazel build //...
bazel test --test_output=errors --verbose_failures //...
bazel test --test_output=errors --verbose_failures -c opt \
  --flaky_test_attempts=3 $(bazel query 'attr("tags", "opt_mode", //...)')
bazel run //bazel:buildifier_check
```

The merge-validation workflow also refreshes checked-in generated files with:

```sh
bazel query 'attr("tags", "auto-generated", //...)' | xargs -r -n1 bazel run
```

## Integration tests

Playwright tests live in `terminal/tests/` and cover the converter, terminal,
password update flow, and text editor. Bazel wraps these tests with
`playwright_matrix_test`, running debug and release/optimized variants:

```sh
bazel test --test_output=errors --verbose_failures //terminal:terminal-integration-test-debug
bazel test --test_output=errors --verbose_failures //terminal:text-editor-integration-test-debug
bazel test --test_output=errors --verbose_failures -c opt //terminal:text-editor-integration-test-release
```

Cargo-side demo integration validation is also part of merge validation:

```sh
./demo/scripts/integration-test.sh target/debug/demo-server
./demo/scripts/integration-test.sh target/release/demo-server
```

## Source map

- `src/terminal/`: terminal UI, xterm.js attachment, terminal tabs, and terminal
  client API wiring.
- `src/text_editor/`: editor manager, folder browser, CodeMirror integration,
  PDF viewer, drag/drop behavior, file I/O, search, notifications, and side
  views.
- `src/converter/`: converter UI, API, service recognizers, and conversion tabs.
- `src/portforward/`: port-forward UI, manager, persisted state, schema, and
  engine.
- `src/frontend/`: menu, remotes UI, resize bars, login, and shared frontend
  controls.
- `src/tiles/`: tile model, tile state, splitting, app selection, and layout.
- `assets/`: HTML bootstrap, JavaScript assets, icons, PDF.js files, and static
  browser assets.
- `tests/`: Playwright integration tests and test configs.
